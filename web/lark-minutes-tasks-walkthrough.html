会开完了，待办散落在逐字稿里。这个 skill 让 Agent 读完妙记或纪要、认出你在会上喊的 Wake Word、列出行动项，再按你勾选的条目真的去约会议、写文档、建飞书任务——不是只生成一串待办字条。

What this repo is for

Action items are buried in transcripts. This skill reads Minutes or meeting docs, recovers your wake-word lines, proposes execution plans, then runs the rows you approve (calendar, Lark docs, tasks) — not only a plain todo list.

装好以后怎么用（与组件路径、模拟器对照）
1 · 放到 commands把 minutes.md 拷到 Claude Code 的 ~/.claude/commands/；宿主只是在磁盘上多了一条命令定义，没有常驻服务。
2 · 准备 Lark CLI本机执行 lark-cli config init 与 lark-cli auth login --domain all；后续拉妙记/文档走的是你已授权的身份。
3 · 开会时喊 Wake Word在会议里说出你预设的触发语；会后 Agent 会从全文里把这段指令捡出来，不用你背原话。
4 · 跑 /minutes把妙记 URL 或 docx 纪要链接交给 Agent；它按 README 中的 6 步逻辑推进，你在编号列表里勾选要它动手的那几项。
After install — how to run it
1 · Command fileCopy minutes.md into ~/.claude/commands/ — static definition, no daemon.
2 · Lark CLIRun lark-cli config init and lark-cli auth login --domain all; fetches use your tenant login.
3 · Wake word in meetingSay your trigger phrase during the call; the agent recovers it from the transcript later.
4 · InvokeType /minutes plus your Minutes or meeting-doc URL (see README), then pick which numbered actions to run.

数据从哪来、边界在哪

Agent 通过 Lark CLI 访问你账号有权读的妙记与文档；不能绕过公司权限「偷」未授权内容。发消息类动作 README 写明可先复制到剪贴板再由你粘贴——实际发送仍在你手上。

Data & safety

Lark CLI only reaches resources your login can read. “Send message” may mean draft + clipboard — you still hit send.

GitHub 上星标不多时，往往说明什么？

  • 需求：盯妙记、要把「会上说了算」落成事的人不少，但真正愿意配置 Lark CLI + 授权 + Agent 宿主的人是小众。
  • 传播：单文件 minutes.md、MIT、双语 README，转发成本低；话题偏「效率工具」而非泛流量。
  • 摩擦：强依赖飞书/Lark 租户与 CLI——环境不配齐就卡在第一步；星少不一定代表不好用，可能是集成门槛过滤了大部分围观收藏。
  • 边界：执行型 Agent 触碰日程、文档、任务，权限与误操作风险要按公司规范自己把控。

右上数字为制作本页时的快照；以 仓库主页 为准。

Reading modest star counts

  • Demand: Minutes → actions resonates with power users; fewer stars often means CLI + tenant setup filters casual stargazers.
  • Ship shape: One-file command + MIT + EN/CN README is easy to try if you already live in Lark.
  • Friction: lark-cli auth is the gate; low stars ≠ low quality.
  • Risk: Execution touches calendars/docs/tasks — keep org policy in mind.

一、常见疑问

FAQ

Q 不用 Claude Code 能用吗？
A 可以：把 minutes.md 放到你的宿主读取命令定义的目录；具体路径因产品而异。
Q Not on Claude Code?
A Still copy minutes.md wherever your host loads slash commands from.
Q Wake Word 会泄露我说了什么吗？
A 处理对象是已有妙记/文档里的文字；谁能读这份妙记，本身就受飞书权限范围约束。不要在会上说超出你职权的敏感指令。
Q Wake-word privacy?
A It mines text you already stored in Minutes/docs under your Lark ACL — keep sensitive verbal orders off the record you don’t own.
Q 和「只生成待办」有什么区别？
A README 强调会执行选中项（约会议、建文档等）；发消息类往往是草稿+剪贴板，最终发送仍由你确认。
Q vs plain todo extraction?
A README promises execution paths; messaging may still be draft+clipboard for safety.
